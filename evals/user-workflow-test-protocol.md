# User workflow test protocol

Status: requires real participants.

Observe technically capable users—and at least one first-time project creator—using their preferred agent. Do not coach away confusion. Record completion, questions answered, time to selected direction, revisions before selection, PRD usefulness, next-session resume, abandonment reason, and whether optional context improved a final direction.

Ask participants to inspect the private profile, discovery artifact, and generated project handoff. Treat privacy concern, unclear command names, weak recommendation, and failure to resume as product findings. Keep any personal data out of the repository.
