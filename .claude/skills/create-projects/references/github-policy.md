# GitHub publication policy

Publishing is a separate explicit approval. Ask for owner or organization, final name, visibility, and initial push after local creation succeeds. Then check `gh auth status`; never inspect or store credentials.

Guide normal `gh auth login` when authentication is missing and check again. Create and push each remote independently. If any operation fails, preserve its local repository, record the exact recovery command, append a journal result, and continue with unrelated projects only when safe.
