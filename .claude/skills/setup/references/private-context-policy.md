# Private context policy

Use `~/.wtfdoibuild/` only for the setup profile, append-only journal, project rollup, out-of-repository briefs, and user-approved résumé copies. Keep the directory outside generated repositories and out of version control.

Never write credentials, API keys, OAuth tokens, calendar exports, telemetry, raw memory, or third-party content that the user did not approve. Store memory permission and source metadata only; reread the original source after consent instead of copying it here. A user can inspect, edit, or remove this directory at any time. Journal entries contain only small user-approved lifecycle facts and reasons.
