# Domain extension scan

Search the target domain before selecting an implementation framework. Run focused web and repository searches for the problem, current products, technical approaches, user complaints, and a mandatory limitations query: “<current approach> limitations”, “<current approach> complaints”, or equivalent.

For each useful source, capture a source card: source URL, audience, current approach, strengths, limitations, and relevance. Derive one differentiation vector that names what the first wedge does differently because of the selected user evidence. Reject a headline that is only table stakes. Cite only tool-returned URLs from this run.
