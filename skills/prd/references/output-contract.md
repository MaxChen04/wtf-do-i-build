# PRD output contract

Use the template headings in order. Every PRD opens with the user-readable digest, conceptual Mermaid diagram, handoff preamble, and resume header. Its detailed sections cover the problem and why this builder, JTBD, tangible product, learning, measurable success and evals, positive non-goals, constraint-derived ADR, decision openings, dependency-ordered build plan, failure table, grounding contract, and references.

Every build-step row has a frozen `P<phase>.<step>[.<substep>]` ID, one imperative, a machine-checkable acceptance criterion, an inline verified reference or local evidence, a `> Teaches:` annotation, and one valid status marker. Do not add dates or host-specific commands.
