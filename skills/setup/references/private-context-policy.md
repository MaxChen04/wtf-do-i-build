# Private context policy

Use `~/.aviator-hamster/` only for the setup profile, append-only journal, project rollup, out-of-repository briefs, and user-approved résumé copies. Keep the directory outside generated repositories and out of version control.

Never write credentials, API keys, OAuth tokens, calendar exports, telemetry, or third-party content that the user did not approve. A user can inspect, edit, or remove this directory at any time. Journal entries contain only small user-approved lifecycle facts and reasons.
